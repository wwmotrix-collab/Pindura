Use a base replicável do Pindura para adaptar este assistente a um novo projeto.

Objetivo:
Reaproveitar cerca de 70% da estrutura, scripts, prompts e regras, criando apenas a camada específica do novo app.

Passos:
1. Identifique stack do novo projeto.
2. Separe regras genéricas das regras específicas.
3. Crie config/project-[nome].md.
4. Crie prompts específicos.
5. Não copie identidade visual Pindura se o novo projeto tiver outra marca.
6. Preserve boas práticas: segurança, debug, git status/diff, sem commits automáticos.
